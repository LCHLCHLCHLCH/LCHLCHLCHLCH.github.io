这个博客是怎么写出来的
发表于 2025 年 6 月 20 日 | 更新于 2026 年 6 月 20 日
总字数: 6313 | 阅读时长: 22 分钟 历史 设计 技术 Hexo xp.css 博客

从一张空白 HTML 页面到一个完整的 Windows XP 风格博客，这个项目经历了两个主要阶段：先是纯手工打造的桌面模拟器，后来又用 Hexo 重建了一套三栏多窗口布局。这篇文章尽可能详细地记录整个过程。

第一季：纯静态桌面模拟

起点：一个空白文件

一切从 index.html 开始 —— 只有 193 个字符，<body> 里面什么都没有，只有 <title>LCH</title>。这是整个项目的第一个 commit。

第一次打开这个页面，浏览器里是一片空白。接下来要做的就是把它变成一个可以点、可以拖、可以打开窗口的 Windows XP 桌面。

概念诞生

XP 桌面模拟的核心 idea 是：把网页做成操作系统的样子。桌面上有图标，双击打开窗口，窗口可以拖动、缩放、最大化，文件夹里能看到文件列表。手机端则是一个简化版，不需要那么复杂的窗口系统，只要能看到文章就好。

于是确定了双页面的架构：

  • **desktop.html**：PC 端（屏幕宽度 ≥ 1024px），完整的 XP 桌面体验
  • **mobile.html**：手机端，简化版文章列表
  • **index.html**：路由器，检测屏幕宽度后自动跳转

UI 框架选型

做 XP 风格界面，从头写 CSS 太费劲了。很快发现了 XP.css —— 一个用纯 CSS 还原 Windows XP Luna 主题的开源库。它提供了 .window、.title-bar、.title-bar-controls、.window-body、.status-bar 等组件，按钮、输入框、树形视图全都自带 XP 风格。

最关键的是标题栏那个蓝白渐变：linear-gradient(180deg, #0997ff, #0053ee 8%, #0050ee 40%, #0066ff 88%, #005bff 95%, #003dd7 100%) —— 这就是 Luna 主题的灵魂。

另一个关键库是 marked.js，用来在浏览器端把 Markdown 文件实时渲染成 HTML。

桌面壁纸：Bliss 渐变

XP 的默认壁纸 Bliss 是那张著名的加州绿丘蓝天照片。但在纯 CSS 下，可以用一个多层渐变来近似还原：

上半部分是蓝天（从深蓝渐变到浅蓝白色），65% 处是地平线，下半部分是绿色山丘。后来在 Hexo 版本里换成了真正的 bliss.jpg 照片。

图标系统

桌面图标的行为经过了几轮打磨：

  1. 第一版：只有 emoji 占位符
  2. 第二版：引入了 Windows XP High Resolution Icon Pack（451 个 PNG 图标），换上了真正的 XP 图标
  3. 交互打磨：XP 的图标交互很特别 —— 悬停不做任何事；单击显示蓝色半透明遮罩 + 白色文字背景；点击桌面空白处取消选中。这些都通过 .selected 类配合 rgba(0,102,204,0.45) 遮罩层实现

最终桌面上的图标有六个：

图标 功能
我的电脑 （预留）
博客文章 打开资源管理器
关于 打开关于页面
Internet Explorer 内嵌浏览器
控制面板 字体大小 / 文字可选设置
GitHub 跳转到作者 GitHub

窗口管理器：整个桌面模拟的核心

这是整个项目里最复杂的部分，大约 250 行纯 JavaScript，实现了：

窗口创建（wm.create）：

  • 动态创建 <div class="window desktop-window"> 元素
  • 每个窗口有独立的标题栏（图标 + 标题 + 最小化/最大化/关闭按钮）
  • 每个窗口在任务栏有对应按钮（后来任务栏被隐藏了）
  • 窗口位置完全随机（带一点偏移避免完全重叠）
  • 支持不可缩放模式（{ resizable: false }），给控制面板用

拖拽系统：

  • 标题栏 mousedown → 记录鼠标相对窗口左上角的偏移 → document mousemove 更新 left/top
  • mouseup 时停止拖拽
  • 修复过一个经典 bug：最大化窗口拖拽时不会自动还原。解决方案是加了一个 pendingMaxDrag 状态：mousedown 记录位置，mousemove 超过 3px 阈值时先调用 toggleMaximize() 还原再开始拖拽，还原后窗口自动定位到鼠标下方

缩放系统：

  • 6px 边缘检测区域，支持 8 个方向：nw / n / ne / w / e / sw / s / se
  • 每个方向对应不同的光标样式和缩放逻辑
  • 向西/向北缩放时同时更新 left/top，保持右下角不动
  • 最小尺寸限制：280×160 px

最大化 / 还原：

  • 双击标题栏切换
  • 最大化时 left: 0; top: 0; width: 100vw; height: 100vh
  • 还原时恢复到之前保存的位置和大小
  • .maximized 类去除窗口圆角

z-index 管理：

  • 全局计数器 zCounter，从 100 开始
  • 每次点击窗口 bringToFront() 递增 z-index
  • 保证最后点击的窗口始终在最前面

资源管理器

双击”博客文章”图标打开的资源管理器，模仿了 Windows XP 的文件浏览体验：

  • 左栏：大图标网格，每个文件一个图标（.md 用传真封面图标，.html 用 HTML 文档图标）
  • 右栏：预览面板，显示选中文件的标题、类型、日期、摘要
  • 状态栏：显示文件数量
  • 交互：单击选中 + 预览，双击打开

文章内容通过 fetch('posts/index.json') 动态加载，然后用 marked.js 渲染 Markdown。这个设计让博客变成了一个模拟的”文件夹”。

控制面板

一个不可缩放的设置窗口，包含两个配置项：

  • 字体大小滑块（6 档，10px–20px）：通过 CSS 自定义属性 --content-font-size 实现，localStorage 持久化
  • 文字可选开关：通过 --content-select 属性切换 text / none，同样持久化

所有窗口内的正文区域都响应这两个 CSS 变量，无需刷新页面。

移动端

手机版走的是完全不同的路线 —— 没有桌面、没有窗口、没有拖拽。就是一个简洁的 XP 风格文章列表：

  • 顶部蓝白渐变标题栏
  • 文章卡片用 XP 凸起边框（inset box-shadow）
  • 点击卡片跳转到 post.html?slug=xxx
  • 底部字体大小下拉框

post.html 是一个全屏窗口阅读器：100vw × 100vh，标题栏 32px，同样用 marked.js 渲染，同样响应字体大小和文字可选设置。

第二季：Hexo 重构

为什么要重构

第一季的桌面模拟虽然交互很酷，但有一些天生的问题：

  • SEO 为零：所有内容靠 JavaScript 动态加载，搜索引擎什么也看不到
  • 没有真正的页面：每篇”文章”本质上是 JS 动态渲染的，没有独立的 URL
  • 维护困难：每次新增文章要手动更新 index.json
  • 桌面模拟很重：窗口管理器 + 拖拽 + 缩放接近 600 行 JS，对静态博客来说太重了

所以决定用 Hexo 重建，保留 XP 风格但去掉桌面模拟，让每篇文章都是一个真正的独立页面。

Hexo 是什么

Hexo 是一个 Node.js 静态站点生成器。它把 Markdown 文章 + EJS 模板 + 配置文件 → 编译成纯静态 HTML 文件。

核心概念：

  • Source：source/_posts/ 下的 .md 文件，带 YAML Front Matter（标题、日期、标签等）
  • Theme：themes/<name>/ 下的 EJS 模板，定义页面结构
  • Generate：hexo generate 一键构建，输出到 public/ 目录
  • Server：hexo server 启动本地开发服务器，支持热重载

三栏多窗口布局

跟第一季的单窗口不同，Hexo 版采用了三栏多窗口布局 —— 每个区域都是一个独立的 XP 窗口：

这个布局参考了一个叫 wenjie-astro 的博客。左边是作者信息，中间是文章正文，右边是目录和最新文章。所有区块都用 XP 窗口包裹，标题栏高度统一 30px（导航栏 32px）。

CSS Grid 布局参数经过多次调整：最初是 220px 1fr 240px，后来扩容到 260px 1fr 280px，最大宽度从 760px → 1100px → 最终 1480px。在 860px 以下折叠为单栏布局。

自定义辅助函数

Hexo 的 EJS 模板本身不提供 TOC、字数统计等功能，于是写了一个 toc-helper.js 来扩展：

  • **toc(content)**：解析文章 HTML 中的 <h1>–<h6> 标签，生成嵌套的 <ul class="tree-view"> 目录树。XP.css 的 tree-view 组件原生支持折叠展开，视觉效果完美
  • **word_count(content)**：统计纯文本字数（去除 HTML 标签），中文按字符数计算
  • **reading_time(content)**：按 300 字/分钟估算阅读时间

TOC 生成是这里面最棘手的 —— 文章里不一定有 <h2>，可能有 <h3> 没有 <h2>，或者 <h2> / <h3> 交替出现。经过三次迭代才正确处理了所有嵌套情况：从第一个出现的标题级别开始编号，兄弟标题之间看级别升降决定是开新子列表还是闭合回到上层。

细节打磨

图标替换：Hexo 版的每个窗口标题栏都有对应的 XP 图标 —— 导航栏用 Internet 快捷方式图标，作者窗口用信息图标，文章用传真封面图标，TOC 和最新文章用 XML 图标。

Bliss 壁纸：从 CSS 渐变换成了真正的 bliss.jpg 照片 —— 就是那张 Charles O’Rear 在加州索诺玛县拍下的绿色山丘和蓝天白云。这是世界上观看次数最多的照片之一。background: url('/images/bliss.jpg') center/cover no-repeat 铺满整个桌面背景。

文章元信息：每篇文章页面显示日期、更新日期、分类、字数、阅读时间、标签，底部有 CC BY-NC-SA 4.0 版权声明。

两个版本对比

第一季（桌面模拟） 第二季（Hexo + xp-blog）
架构 纯静态 HTML/CSS/JS Hexo 静态站点生成器
页面数 4 个 HTML 文件 28 个生成页面
JS 体积 ~600 行（窗口管理器） ~80 行（辅助函数）
SEO 无 完整
文章管理 手动维护 JSON Front Matter + hexo new
布局 桌面 + 浮动窗口 三栏多窗口 Grid
桌面模拟 完整（拖拽/缩放/最大化） 无
移动端 独立 mobile.html 响应式 CSS 折叠
壁纸 CSS 渐变 真正的 bliss.jpg

两个版本各有各的好：桌面模拟版交互极其有趣 —— 你可以拖窗口、缩放、双击最大化，像个真正的操作系统。Hexo 版则更像一个”正经”博客 —— 有独立 URL、有 RSS、搜索引擎能收录。

技术演进线

数字总结

  • Git commits：30+
  • 创建的文件：40+（跨越两个版本）
  • 手写 JavaScript：桌面版 ~600 行 + 移动版 ~80 行 + Hexo 辅助函数 ~100 行
  • 手写 CSS：桌面版 ~350 行 + Hexo 主题 ~370 行
  • 图标资源：来自 Windows XP High Resolution Icon Pack（451 个图标中精选了 16 个）
  • 文章数量：从 3 篇测试文章起步，到现在 5 篇（含本篇）
  • 两季对话总长度：约 29,000 行（第一季 17,685 行 + 第二季 11,711 行）

从一张空白 HTML 到一个完整的 XP 风格博客，中间经历了桌面模拟的狂欢和 Hexo 的重构。回头看看整个过程，最有意思的不是最终的页面效果，而是那些迭代中的细节：标题栏高度从 21px 调到 30px，图标选中遮罩的颜色调了多少次，TOC 嵌套逻辑改了三版才正确，最大化窗口拖拽还原的那个 3px 阈值……

Keep it simple. 但那是在你搞清楚什么不该简单之后。
